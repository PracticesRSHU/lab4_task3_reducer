Етапи реалізації :
1) Створення store.
    Store — це об'єкт що дозволяє зберігати інформацію про стан додатка (сховище) 
    а також містить кілька методів роботи зі станом:
    - метод отримання даних зі сховища;
    - зміна даних за допомогою dispatch;
    - здійснення підписки на зміни стану.
    Створення store здійснюється за допомогою функції createStore(),
    яка є частиною модуля redux, а тому краще буде попередньо імпортувати її.
2) Створення reducer.
    Reducer — це чиста функція, яка вираховує наступне значення
    стану на основі його попереднього значення та застосованої до нього події.
    Стиль оформлення функції може бути будь-яким
    (в наведеному прикладі запропоновано стрілочний синтаксис), 
    але вона приймає два параметри: state та action
        const reducer = (state, action) => {
        };
3) Забезпечення доступності store в react-компонентах.
    Для цього скористаємося модулем react-redux. 
    Здійснювати весь імпорт не має сенсу,  із нього лише можна використати
    призначений для вирішення конкретного завдання вбудований <Provider>
4) Робота зі змінною стану.
    Для отримання даних потрібно застосувати хук useSelector(), 
    який є частиною модуля react-redux.
    
    Для оновлення даних ідеологія redux передбачає залучення диспетчера, який виконує:
    - передачу до reducer назви actions;
    - передачу даних, необхідних для зміни стану.
    Створення dispatcher здійснюється за допомогою хука useDispatch(),
    який  є частиною модуля react-redux
